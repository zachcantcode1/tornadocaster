About the Project

NADOCast-First Severe Weather Guidance

A product layer for published NADOCast probability grids: custom maps, summaries, local lookups, overlays, and run-to-run interpretation.

What is Tora Weather?

Tora Weather is a severe weather guidance viewer built around NADOCast probability grids. Instead of recreating the full machine-learning forecast pipeline, the active code treats published NADOCast GRIB2 files as the source of truth and focuses on the product layer around them.

The goal is to make those grids easier to use locally: render clear CONUS maps, normalize probability values, summarize active risk areas, sample nearest grid points, and compare runs as new guidance arrives.

Data Sources

The primary input is public NADOCast GRIB2 output fetched from data.nadocast.com. The current workflow resolves matching forecast files, downloads the selected hazard grid, and decodes it with cfgrib and xarray.

Supported hazard tokens include tornado, sig_tornado, hail, sig_hail, wind, sig_wind, and wind_adj. NADOCast percent grids are normalized into 0.0-1.0 probabilities before analysis and rendering.

How the Forecast Pipeline Works

01

Resolve a NADOCast run

Find the latest matching run, or use an explicit date, cycle, and hazard requested from the command line.

02

Fetch the public GRIB2 grid

Download the selected probability file from data.nadocast.com instead of building a forecast from raw model fields.

03

Decode with cfgrib and xarray

Open the GRIB2 data into structured arrays that can be sampled, summarized, and plotted.

04

Normalize probabilities

Convert NADOCast percent grids into 0.0-1.0 probabilities for consistent analysis.

05

Analyze the grid

Generate probability summaries and nearest-point lookups for local interpretation.

06

Render the map

Plot a CONUS probability map using the existing SPC/NADOCast-style color ramp.

07

Interpret run-to-run changes

Future work should add source adapters and overlays around this simpler product layer.

Current Scope

The active application fetches NADOCast GRIB2 files, decodes probability grids, normalizes percent values, renders CONUS maps, and provides small helpers for summaries and point sampling.

The previous model-building and training stack has been removed from the active code path. Tora Weather is now a viewer and interpretation layer for existing NADOCast guidance, not a replacement forecast model.
